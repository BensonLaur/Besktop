# .agent-skills - 项目专属 Agent 技能系统

## 这是什么

一套让 AI Agent 按需加载项目专属工作流程的机制。解决的核心问题：**项目指令文件（如 AGENTS.md、CLAUDE.md）随项目成长变得臃肿，每次会话都要消耗大量 token 加载很少用到的详细流程。**

本 README 是机制说明，不是某个具体技能文档。它应尽量适用于所有项目，并随 `.agent-skills/` 一起自传播。

## 设计理念

```text
+-------------------------------------------------+
|  项目指令文件（第一级 · 每次会话加载）             |
|  例如 AGENTS.md / CLAUDE.md / 其他强制加载文件     |
|  +---------------------------------------------+ |
|  |  触发词映射表（几行文本，极少 token）          | |
|  +------------------+--------------------------+ |
+---------------------+---------------------------+
                      |
                      | 用户说出触发词时
                      v
+-------------------------------------------------+
|  .agent-skills/<name>.md（第二级 · 按需加载）     |
|  +---------------------------------------------+ |
|  |  完整的分步流程、命令、路径、历史数据          | |
|  +---------------------------------------------+ |
+-------------------------------------------------+
```

**原则：**

- **第一级极简**：项目指令文件只放触发词到技能文件的映射表。
- **第二级详尽**：技能文档包含完整执行步骤、命令、验证条件，Agent 读完即可独立执行。
- **按需加载**：只在用户说出触发词时读取对应文档，平时不消耗 token。
- **独立自洽**：每个技能文档自包含所有上下文，不依赖 Agent 的记忆或对话历史。

## 项目指令文件

不同 Agent 使用的第一级入口可能不同：

- Codex 项目优先使用 `AGENTS.md`。
- Claude 项目通常使用 `CLAUDE.md`。
- 其他 Agent 可以使用各自强制加载的项目说明文件。

无论入口文件叫什么，都只维护轻量触发表；完整流程写入 `.agent-skills/*.md`。

## 如何在新项目中实施

### 第 1 步：创建目录并放入本文档

在项目根目录创建 `.agent-skills/` 目录，并将本文档复制为 `.agent-skills/README.md`。

这确保了每个实施此机制的项目都自带完整说明。未来打开任何一个项目，读取 `.agent-skills/README.md` 即可回忆全部设计理念，无需从外部查找。

### 第 2 步：在项目指令文件中添加触发机制

在 `AGENTS.md`、`CLAUDE.md` 或其他 Agent 强制加载的项目指令文件中添加类似内容：

```markdown
## 项目技能 (.agent-skills/)

当用户提到以下关键词时，先读取对应技能文档，按文档中的步骤执行，不要凭记忆操作。

| 触发词 | 技能文件 | 说明 |
| --- | --- | --- |
| "xxx"、"yyy" | `.agent-skills/xxx.md` | 一句话说明 |

> 技能文档是延迟加载的，仅在触发时读取，平时不消耗 token。
```

### 第 3 步：编写技能文档

在 `.agent-skills/` 下创建 `.md` 文件，遵循以下模板：

```markdown
# 技能: <名称>

## 概述

一句话说明这个技能做什么。

## 前置条件

执行前需要满足的条件（环境、文件、构建状态等）。

## 执行步骤

### 步骤 1: <动作>

具体命令和说明。

### 步骤 2: <动作>

...

## 验证步骤

说明如何确认关键操作已经成功。

## 注意事项

易错点、边界情况、历史教训、异常处理。
```

### 第 4 步：在触发表中注册

在项目指令文件的映射表中加一行。触发词只维护在第一级入口文件中，避免 README 和入口文件重复维护两份总表。

## 技能文档编写原则

1. **可独立执行**：Agent 仅凭文档内容就能完成全部操作，无需额外上下文。
2. **命令可复制**：所有 shell 命令使用完整绝对路径，不依赖当前工作目录假设。
3. **包含验证步骤**：每个关键操作后说明如何确认成功。
4. **记录历史数据**：如果涉及对比（如性能基准、版本基准、已知坑），在文档中保留。
5. **说明异常处理**：写明常见失败场景和恢复方法。

## 目录约定

- 目录名固定为 `.agent-skills/`（点前缀，不干扰项目目录）。
- 技能文件用语义明确的英文命名，2 到 4 个词为宜，看到文件名就能联想功能，例如 `collect-release-notes.md`、`publish-delivery.md`、`build-and-test.md`。
- 本 `README.md` 是机制说明，不是技能文档，不需要在触发表中注册。
- 具体技能列表由项目指令文件中的触发表维护；README 不重复维护完整触发表。

## 编码约定

- 如果技能文档包含中文，并且项目主要在 Windows、PowerShell、Visual Studio 环境使用，推荐使用 UTF-8 BOM。
- UTF-8 BOM 可以避免 Windows PowerShell 5.1、Visual Studio 或旧式工具按本机 ANSI/GB2312 误读导致乱码。
- 用 PowerShell 读取这些文档时，优先显式指定 `-Encoding UTF8`；新增或重写技能文档后，可以检查文件头是否为 `EF BB BF`。
- 本项目约定：`AGENTS.md`、`.agent-skills/*.md`、`version.txt` 统一使用 UTF-8 BOM。

## 快速参考：从零开始

当用户在一个新项目中提供本文档并说“参考这个文档，创建技能系统”时，Agent 应该：

1. 创建 `.agent-skills/` 目录，将本文档写入 `.agent-skills/README.md`（自传播：确保每个项目都携带完整说明）。
2. 在项目指令文件中添加“项目技能”章节和触发表框架。
3. 根据用户描述的触发词和流程，创建第一个技能文档。
4. 在触发表中注册该技能。
