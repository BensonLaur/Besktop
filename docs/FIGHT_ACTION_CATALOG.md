# Icon Fight 格斗动作候选库

本文档记录 Besktop Icon Fight 可采用的格斗动作、统一命名和第一轮实现范围。它服务于公开 Core 的基础玩法，不包含 Plus 私有演出脚本、付费动作包或商业文案。

动作参考来自拳击、踢拳和跆拳道的公开规则与教练资料，但 Besktop 的目标不是模拟专业比赛，而是把动作转写为“图标薄片 + 白色两段式四肢”能够清楚表达的桌面演出。

## 筛选原则

候选动作优先满足：

- 在 48–96px 图标尺寸下，仅看轮廓也能辨认。
- 能使用当前肩—肘—手、胯—膝—脚两段式骨架表达。
- 有明确准备、发力、接触和回收节奏。
- 能与闪避、格挡、受击或打空组成互动，而不是孤立播放。
- 不依赖手指、面部表情、复杂贴身缠斗或地面关节结构。
- 不要求增加大量全屏透明层、粒子或重复图标绘制。

## 参考资料

- [England Boxing Coaching Handbook Part 1](https://www.englandboxing.org/wp-content/uploads/2026/02/EB_Boxing-Coaching-Handbook-Part-1_09_02_26.pdf)：直拳、勾拳、上勾拳，以及格挡、拨挡、后仰、侧闪、下潜、摇闪、侧步和防守后反击。
- [IBA Coach Manual](https://www.iba.sport/documents/iba-coach-manual/)：直拳、勾拳和上勾拳的基础分类，以及前后手、头部和身体目标变化。
- [WAKO Kickboxing Rules](https://www.wako.sport/fr/_files/ugd/6445bf_4945eef1b6a649b7bf055956396dace0.pdf)：前踢、侧踢、回旋踢、后跟踢、月牙踢、下劈踢、跳踢、旋转后踢和扫腿等候选动作。
- [World Taekwondo Competition Rules](https://www.worldtaekwondo.org/att_file/documents/WT%20Competition%20Rules%20and%20Interpretation%20%28June%2014%2C%202024%2C%20GA%20E%20Vote%29.pdf)：转身后踢要求头、肩和身体随踢击一起转动，可作为薄片转身阶段的动作参考。

## 新闻参考动作的统一命名

用户提供的新闻画面包含四种核心表现。产品文案可以保持口语化，代码和动作数据使用稳定 ID：

| 画面称呼 | 动作 ID | 说明 |
| --- | --- | --- |
| 挥拳 | `swing_punch` | 大幅度摆臂的夸张挥拳，强调新闻画面的辨识度；不同于短距离紧凑勾拳。 |
| 侧踢 | `side_kick` | 收膝、支撑脚稳定、脚向身体侧面推出。 |
| 旋转 + 侧踢 | `spinning_back_kick` | 技术表现更接近转身后踢；头肩先转、图标薄片经过侧面、后腿向目标推出。 |
| 上身后躲 | `layback` | 脚下尽量保持位置，上身和图标薄片快速后仰，再恢复站姿。 |

这四项是第一轮范围的一部分，但“参考动作”不等于当前 Core 已经完成对应可交互实现。

## 攻击动作候选

### 拳法

| 动作 | 动作 ID | 视觉特征 | 优先级 |
| --- | --- | --- | --- |
| 前手直拳 | `lead_straight` | 前手快速伸出并立即收回，适合试探。 | 第一轮 |
| 后手直拳 | `rear_straight` | 后肩和图标薄片随拳转动，力量感强。 | 第一轮 |
| 上勾拳 | `uppercut` | 身体先下沉，拳头由下向上穿过中线。 | 第一轮 |
| 紧凑横勾拳 | `hook` | 肘部弯曲，拳头走短弧线，区别于夸张挥拳。 | 第一轮 |
| 夸张挥拳 | `swing_punch` | 蓄力更长、弧线更大，保留新闻参考表现。 | 第一轮 |
| 过顶摆拳 | `overhand` | 身体向前压，拳头从上方向下砸落。 | 上线后 |
| 旋转背拳 | `spinning_backfist` | 图标薄片转身后用后手横扫，轮廓醒目。 | 上线后 |
| 肘击 | `elbow_strike` | 贴身、幅度短，依赖双方距离关系。 | 长期 |

### 腿法与膝法

| 动作 | 动作 ID | 视觉特征 | 优先级 |
| --- | --- | --- | --- |
| 前踢/蹬踢 | `front_kick` | 收膝后向前推出，轮廓清楚、回收直接。 | 第一轮 |
| 侧踢 | `side_kick` | 身体侧向目标、支撑腿稳定、踢腿水平推出。 | 第一轮 |
| 回旋踢 | `roundhouse_kick` | 髋部和支撑脚旋转，小腿横向扫过目标。 | 第一轮 |
| 转身后踢 | `spinning_back_kick` | 头肩先转，薄片经过侧面，后腿直线推出。 | 第一轮 |
| 低扫腿 | `low_kick` | 攻击对方支撑腿，需要配套失衡反馈。 | 上线后 |
| 旋转勾踢 | `spinning_hook_kick` | 腿伸出后用脚跟横向回勾，弧线感强。 | 上线后 |
| 下劈踢 | `axe_kick` | 腿先高举，再从上向下劈落。 | 上线后 |
| 膝击 | `knee_strike` | 近距离抬膝撞击，适合短促重击。 | 上线后 |
| 飞膝 | `flying_knee` | 跳起、前移、抬膝，需要可靠落地和边界控制。 | 长期 |
| 扫腿放倒 | `foot_sweep` | 扫过脚踝使对方失衡，需要倒地/起身系统。 | 长期 |

## 防守、闪避与位移候选

| 动作 | 动作 ID | 视觉特征 | 优先级 |
| --- | --- | --- | --- |
| 上身后仰 | `layback` | 脚下保持，薄片快速后仰避开直线攻击。 | 第一轮 |
| 左右侧闪 | `slip_left` / `slip_right` | 身体和图标薄片偏离攻击线，手仍可反击。 | 第一轮 |
| 格挡/拨挡 | `parry` | 前臂短促抬起，将来拳拨离中线。 | 第一轮 |
| 双臂护头 | `high_guard` | 两臂覆盖图标上半部，适合承受重击。 | 上线后 |
| 下潜 | `duck` | 身体整体降低，避开高位拳。 | 上线后 |
| U 形摇闪 | `roll` | 下潜后从另一侧起身，适合接勾拳反击。 | 上线后 |
| 侧步 | `side_step` | 整体向攻击线侧面移开并保持朝向。 | 上线后 |
| 枢轴转位 | `pivot` | 以一脚为轴改变双方夹角。 | 长期 |
| 抬腿防扫 | `leg_check` | 抬膝并转动小腿，抵挡低扫。 | 长期 |

## 受击、打空与结果动作

攻击只有与结果反馈配对后才会像一次互动。第一轮至少需要：

| 结果 | 动作 ID | 用途 |
| --- | --- | --- |
| 轻击偏转 | `light_hit_react` | 薄片快速偏转、局部后移后恢复，不中断太久。 |
| 重击后退 | `heavy_stagger` | 身体明显后仰并滑退一至两步，产生重击反馈。 |
| 打空回收 | `whiff_recovery` | 攻击越过目标后短暂失衡，再收回防守姿态。 |

后续候选包括单脚失衡、旋转踉跄、坐倒、仰倒、短暂眩晕、撑地起身、胜利庆祝和挑衅。倒地前必须先建立稳定的层级、碰撞和起身空间管理。

## 第一轮冻结范围

第一轮只实现以下动作族，不在同一轮加入抱摔、地面缠斗或复杂粒子：

### 新闻参考动作

1. `swing_punch`
2. `side_kick`
3. `spinning_back_kick`
4. `layback`

### 新增基础动作

1. `lead_straight`
2. `rear_straight`
3. `uppercut`
4. `hook`
5. `front_kick`
6. `roundhouse_kick`
7. `slip_left` / `slip_right`，作为一个可镜像动作族实现
8. `parry`

### 必须配套的结果反馈

1. `light_hit_react`
2. `heavy_stagger`
3. `whiff_recovery`

第一轮范围的目标不是一次性让所有图标混战，而是建立能组合出“试探、进攻、防守、反击、命中、格挡、闪避和打空”的基础动作语言。

## 推荐互动组合

| 进攻 | 防守/结果 | 可衔接反击 |
| --- | --- | --- |
| 前手直拳 | 拨挡、左右侧闪、轻击偏转 | 拨挡接前手直拳；侧闪接勾拳 |
| 后手直拳 | 上身后仰、拨挡、重击后退 | 后仰接后手直拳 |
| 上勾拳 | 后撤、轻击偏转 | 上勾拳命中接重击后退 |
| 勾拳/挥拳 | 下潜、后仰、打空回收 | 下潜后接上勾拳 |
| 前踢 | 侧步、格挡、重击后退 | 格挡后接直拳 |
| 侧踢 | 后撤、侧闪、重击后退 | 侧闪后接侧踢 |
| 回旋踢 | 后仰、格挡、重击后退 | 格挡后接后手直拳 |
| 转身后踢 | 侧步、打空回收、重击后退 | 打空后给对手反击窗口 |

## 本轮不做

- 抱摔、过肩摔、锁技和地面缠斗。
- 多演员同时压在一个目标上的围攻动作。
- 依赖复杂布料、面部或手指的动作。
- 大量全屏闪光、持续粒子和屏幕震动。
- 为实现新动作而降低默认演员数量或删除真实图标显示。
