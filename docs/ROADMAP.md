# Besktop 路线图草案

路线图按“先文档，后原型，再产品化”的顺序推进。

## 阶段 0：文档和边界

目标：把项目方向写清楚，避免边做边改大方向。

- 明确产品定位：桌面娱乐小工具，首个玩法 Icon Fight。
- 明确开源与商业关系：Core 开源，Plus 私有商业扩展。
- 明确发布形态：用户下载一个 `Besktop.exe`，Plus 是内部扩展层。
- 明确文档语言：项目文档默认使用中文。
- 明确提交规范：使用约定式提交，摘要使用中文。
- 明确技能系统：通过 `AGENTS.md` 和 `.agent-skills/` 记录长期流程。

## 阶段 1：感染演出视觉 MVP

目标：证明“桌面舞台 + 图标演员 + 基础动作”能在 Windows 上自然呈现。先让图标演员可信地长出手脚、走路和转身，再进入打架动作。

- 将 GUI `besktop` 从 MessageBox 壳改为 Win32 全屏应用壳。
- 创建覆盖主显示器和任务栏的桌面舞台窗口。
- 支持 `Esc` 退出，预留强制退出热键。
- 建立 `DesktopSnapshot` 数据模型。
- 重绘壁纸、图标、图标文字和任务栏静态视觉。
- 给至少两个图标演员添加白色手脚。
- 实现 Locomotion MVP：左右巡逻、转身、脚落地、手脚反相摆动。
- 在走路模型成立后，实现拳击、转身侧踢、闪避、受击四类关键动作。
- 实现图标本体的双面薄片翻转：左转、右转仍显示正常图标，侧面对屏幕时接近一条窄线。
- 实现局部 3D 两段式手脚骨架，让手脚像长在图标薄片外侧，而不是贴在同一个平面上。

## 阶段 2：真实桌面舞台采集增强

目标：从 demo actor 过渡到稳定读取和复刻真实桌面舞台。

- 通过 `IDesktopWallpaper` 获取壁纸路径和显示方式。
- 当前 MVP 通过桌面 `SysListView32` 只读采样读取图标位置和名称。
- 后续替换或补强为 Shell View / `IFolderView` 与 Shell 图像接口。
- 由 Besktop 绘制图标文字，支持抖动、错位、淡出。
- 截取任务栏静态视觉。
- 处理高 DPI。
- 初步处理多显示器，至少保证主显示器稳定。
- 处理 Explorer 重启、桌面刷新、图标变化。
- 建立壁纸、图标、文字、任务栏采集失败时的降级路径。

## 阶段 3：免费体验打磨

目标：让免费版足够好玩，愿意被传播。

- 增加基础动作组合。
- 将 MVP 中硬编码动作迁移为轻量 `AnimationClip`，包含关键姿态、IK 目标、落地事件、命中事件和 root motion。
- 增加短时间自动导演逻辑。
- 增加轻量设置项。
- 增加录屏友好的演出模式。
- 增加自愿打赏入口。

## 阶段 4：Plus 商业闭环

目标：在不影响免费传播的前提下，加入付费高级玩法。

- 定义内部玩法包格式。
- 支持签名玩法包或签名内置资源。
- 接入兑换码激活。
- 支持本地授权验证。
- 与 Besktop-Plus 的支付授权服务打通。
- 验证 Besktop-Plus 是否能同步 Core 并打包出最终单 exe。
