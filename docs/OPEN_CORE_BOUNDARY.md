# Besktop 开源与商业边界

Besktop 采用 open-core 思路：开源核心必须真有乐趣，商业部分负责更丰富、更社交、更个性化的扩展。

## 基本原则

Core 不是残缺试用版。用户即使不付费，也应该能获得完整的基础 Icon Fight 体验。

Plus 的价值应来自额外惊喜：

- 更适合情侣、朋友、社交传播的互动玩法。
- 更多动作、台词、场景和可配置内容。
- 更方便录屏、分享和定制的能力。
- 商业授权、支付、售后和私有资源维护。

用户看到的产品只有一个：`Besktop`。Plus 是商业扩展层，不是第二个独立软件。

## Besktop Core

Core 放在公开仓库，使用 GPL-3.0。

Core 可以包含：

- 主程序 `Besktop.exe` 的公开构建逻辑。
- 安全桌面舞台。
- 基础桌面图标扫描。
- 基础图标演员模型。
- 基础动作：待机、移动、出拳、踢腿、闪避、受击、恢复。
- 免费基础玩法内容。
- 玩法包 / 插件的公开数据结构和加载接口。
- 授权验证客户端的公开接口和公钥验证逻辑。
- 用于开发和测试的 mock 授权、示例包和伪数据。
- 托盘控制、设置和安全退出。
- 自愿打赏入口。
- 公开文档和开发计划。

Core 不应包含：

- 支付平台密钥。
- 授权私钥。
- 客户订单或邮箱数据。
- Plus 私有资源。
- 付费玩法源码、未公开素材或商业脚本。
- 真实商户号、真实回调地址中的密钥参数。
- 生产授权服务的私有实现细节。

## Besktop Plus

Plus 放在私有仓库，用于商业化。

Plus 可以包含：

- 情侣互动包。
- 朋友互动包。
- 自定义名称、台词和角色关系。
- 录屏传播增强。
- 付费包资源和签名。
- 支付回调服务。
- 兑换码、授权和设备绑定逻辑。
- 售后工具和订单查询流程。
- 商业发布脚本：同步指定版本 Core，嵌入 Plus 内容，产出最终 `Besktop.exe`。

## 公开仓库应暴露什么

公开仓库应该让用户和未来的自己清楚知道：

- Besktop 是什么。
- 免费版能玩到什么。
- 为什么它不会破坏桌面文件。
- 技术上大致如何实现桌面舞台和图标演员。
- 玩法包 / 插件接口的大致结构。
- 授权验证只在客户端验证签名，不包含私钥。
- Plus 是可选商业扩展，不影响免费版基本乐趣。

公开仓库不需要暴露：

- 具体收入、订单、支付平台密钥和商户配置。
- Plus 包的完整商业动作脚本。
- 私有素材和宣传资源。
- 生产 Worker 代码中涉及平台密钥和业务策略的部分。
- 客户售后数据和订单数据。

## 贡献策略

早期不接受代码 PR，避免未来双授权和商业分发时出现版权边界问题。

欢迎非代码参与：

- bug 反馈。
- 兼容性报告。
- 动作创意。
- 产品命名建议。
- 付费包玩法建议。
- 文档和说明文字建议。

如果未来开放代码贡献，需要先设计贡献者协议或明确版权授权规则。
